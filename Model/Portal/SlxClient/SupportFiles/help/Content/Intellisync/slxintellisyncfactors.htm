Understanding Factors Affecting Synchronization

The time it takes to synchronize data between Sage SalesLogix and Microsoft Outlook or Lotus Notes depends on several factors. The most important factor is the volume of data being passed between the systems. You can improve performance by controlling the volume of data transferred through Intellisync for SalesLogix. There are several ways to do this.

Last Synchronization Date

When data exchange occurs, the date is recorded in a log file. Subsequent synchronizations use this date to check for modifications since the last sync date and only transfer data that has changed since that date. By synchronizing regularly, you can reduce the volume of data that needs to be transferred during any given cycle.  

Security

Synchronization is affected by the built-in data security associated with the user who is logged on to Sage SalesLogix. The amount of data available to transfer is limited by the security rights of the user logged in.

Un-mapping Fields

Intellisync for SalesLogix allows a user to define how the data is mapped between applications. By removing mapping from a field, the translator ends data transfer in that field. Additionally, mapping a previously un-mapped field starts synchronizing the data in that field between applications. See Understanding Intellisync for SalesLogix Field Mapping for additional information.
