Running Contacts and Activities Synchronization

When the Intellisync for SalesLogix configuration is complete in the Sage SalesLogix Web Client (as explained in "Configuring Intellisync for SalesLogix"), changes to contacts and activities are synchronized between Sage SalesLogix and Microsoft Outlook or Lotus Notes according to the in the Sage SalesLogix Web Client (as explained in "Configuring Intellisync for SalesLogix ").

During a synchronization cycle, Intellisync compares records in Sage SalesLogix to those in Microsoft Outlook or Lotus Notes, and makes changes to ensure both systems contain the same information. For example, if a record has been added to one system and not to the other, that record is added to the incomplete system.

Intellisync maintains a log file which includes the number of records changed, added, deleted, or in conflict in Sage SalesLogix and Microsoft Outlook or Lotus Notes. The first time a particular set of data is synchronized, there is no log file. Data from the two systems is simply merged, and any data conflicts between the two systems are resolved using the conflict resolution rules established when configuring Intellisync.

The log file is only generated for automatic synchronizations. To view the log, right-click the Intellisync icon in the System Tray section of the Windows Taskbar and select Show Push Sync Log.

Subsequent synchronizations only include changed, added, or deleted records.  You can initiate a full synchronization by changing the synchronization settings, or selecting Sync Now from the Intellisync main screen or the icon right-click menu.

During a manual synchronization, a confirmation dialog box appears with details about conflicts and/or changes to be made to the data in Sage SalesLogix and Microsoft Outlook or Lotus Notes.

 

Simultaneously synchronizing the Remote Client to the main office database and Microsoft Outlook or Lotus Notes, is not recommended. Doing so significantly reduces synchronization speed and causes memory problems. Sage SalesLogix recommends that you synchronize with the Outlook or Lotus Notes application before or after synchronizing your remote database.
