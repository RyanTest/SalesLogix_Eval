Tracking a Defect

Tracking a defect means you are working to resolve the defect. Use the following questions or refer to your company policies for the business rules for tracking defects in your organization.

  • Do I need to associate a ticket to this defect?

If yes, then create a task for each type of work performed.

  • Do I need to track the time spent working on this defect?

 

If yes, then create a task for each type of work performed.

  • Do I need to change the status of the defect (for example, from Open to In Progress)?

If yes, change the Status box in the Defect Detail view.

  • Do I need to add a resolution for the defect?

If yes, add the resolution on the Defect Detail tab.
