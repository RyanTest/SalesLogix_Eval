Rules for Automatically Merging Records During Import

 

During a lead import you have the option to automatically merge duplicate records that are found using the match filters you select. A duplicate lead may be merged with an existing contact or lead. The records are merged according to the following rules:

  • If the Conflicts option is set to Existing Record Wins:
  • The records are merged and none of the imported lead information is used. This also applies to account fields.
  • If the imported record has data in a field, and the existing record does not, the field in the existing record will remain empty.
  • If the Conflicts option is set to Imported Record Wins:
  • The imported lead information will replace the target (lead, contact or account) record information.
  • If a field in the imported lead record is empty , but the field in the existing record contains information, then the existing record information is saved. This also applies to account fields.
  • When merging to another lead, the address information for the lead being imported will replace the existing lead's address information. A note containing the existing record's address will be created and added to the notes history for that lead.
  • The address record for the lead being imported will never replace an address for a contact or account. A new address will be created using the imported lead's address information. The new address will not be the primary address for the account or contact and the primary and mailing addresses for the account and contact will not be changed.
